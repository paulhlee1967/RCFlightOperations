Installation & Setup

This page is for the person who installs RC Flight Operations — your club's webmaster or the person with server access. It covers requirements, database setup, and first login.

Requirements

RequirementVersionNotes
PHP8.0 or higherPDO and PDO_MySQL extensions required
MySQL / MariaDB5.7+ / 10.3+MariaDB 10.x (common on cPanel) works great
Web serverApache or NginxApache with mod_rewrite, or PHP's built-in server for local dev
ComposerAny recent versionRequired to install Dompdf (PDF export)
Storage~50 MB + uploadsMember photos stored in uploads/

The free tier on most shared hosting providers (Hostinger, SiteGround, etc.) meets all these requirements. If your club already pays for web hosting, RC Flight Operations will likely run there at no extra cost.

Local Development (Mac)

Full details are in LOCAL_DEV.md in the project root. The quick version:

  1. Install Homebrew, then run:
    brew install php mysql composer
  2. Start MySQL and create the database:
    mysql.server start
mysql -u root -e "CREATE DATABASE flightops CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"
mysql -u root flightops < schema_full.sql
  3. Copy config.php.example to config.php and fill in your local database credentials.
  4. Install Composer dependencies:
    composer install
  5. Set the admin password:
    php scripts/set_password.php
  6. Start the built-in PHP server from the project root:
    php -S localhost:8080
  7. Open http://localhost:8080/login.php and log in with admin@yourclub.local and the password you set.

cPanel Shared Hosting Deployment

Full details are in DEPLOY.md (section Moving local data to cPanel). The summary:

  1. Export your local database (if migrating from local dev):
    php scripts/export_db_for_cpanel.php
    This produces export_for_cpanel.sql ready for import.
  2. Upload the project files via FTP or cPanel File Manager into a folder on your host (e.g., public_html/flightops/). Do not upload your local config.php.
  3. Create a MySQL database and user in cPanel → MySQL Databases. Note the database name, username, and password — they'll have your cPanel username as a prefix (e.g., youruser_flightops).
  4. Import the SQL file via phpMyAdmin (Databases → your DB → Import).
  5. Create config.php on the server with your cPanel database credentials (copy from config.php.example and edit).
  6. Install Composer dependencies on the server via SSH:
    cd /path/to/flightops && composer install
    Or upload your local vendor/ folder if SSH isn't available.
  7. Set the admin password if not already set:
    php scripts/set_password.php

First Login

Navigate to https://yourdomain.com/flightops/login.php. Log in with:

  • Email: admin@yourclub.local (change this after login)
  • Password: the one you set with set_password.php

After first login, go to Administration → Users and update the admin email to your real address.

Next Steps After Install

  1. Update club name, logo, and colors in Administration → Configuration → Design.
  2. Set dues rates in Administration → Configuration → Dues Rules.
  3. Add your board members as users in Administration → Users.
  4. Import existing member records via Members → Import.
  5. Design your ID badge template in Administration → Badge design.
  6. Run the Membership Snapshot report to confirm your data looks right.

Maintenance Scripts

Run these from the project root via command line:

ScriptWhat it does
php scripts/set_password.php Interactively set or reset the admin account password. Use this if you get locked out.
php scripts/verify_db.php Checks that your live database schema matches schema_full.sql. Run after pulling updates.
php scripts/export_db_for_cpanel.php Exports a clean SQL dump compatible with cPanel's phpMyAdmin import (strips local-only settings).
