Membership applications

When your club collects membership applications on its website (via WPForms), submissions land in Applications for staff review before anyone is added or updated in the member roster.

Payments stay on the website RC Flight Operations shows what the applicant paid on the form for review. It does not charge cards or auto-post website payments to the ledger. After approval, use Process Signup / Renewal to record dues the same way you would for any other member.

Overview

Each website submission creates a pending application row. Staff compare the form data to existing members (AMA number, name, email, birthday), review uploaded file links (AMA/FAA verification on the website), and decide whether to approve or reject. Approved applications create a new member or update a matched member, copy the badge photo when one was submitted, then hand off to renewal recording.

The navbar badge and dashboard “Needs attention” card show how many applications are still pending — counted across all renewal years, not just the year currently shown in the list.

Who can use it

  • Administrator and Membership Manager — full review, approve, and reject.
  • Club Staff — can review and approve (same as renewal processing access).
  • Report Viewer — no access to Applications.

Review workflow

  1. Open Applications from the top nav (or follow the dashboard link).
  2. Select an applicant from the list to open the detail panel.
  3. Check the suggested member match and any Changes vs matched member table.
  4. Review payment info and uploaded AMA/FAA links (badge photo is handled on approve).
  5. Adjust Renewal type and Renewal year if needed.
  6. Click Approve & continue to recording or Reject.

Approval redirects to Process Signup / Renewal for the member so you can record payment and print a badge. If a badge photo was on the application, it should already be on the member record — go straight to fulfillment when ready. See Renewals & Dues and Badge printing.

Filters & pagination

The list is designed for ongoing season work, not an endless archive of every submission.

ControlBehavior
Status tabs Pending (default), Approved, Rejected, or All
Renewal year Defaults to the club’s current renewal year (same logic as Process Signup / Renewal). Choose All years to browse history.
Search Name, email, or WPForms entry number
Reset Clears search and returns to the current renewal year
Pagination 50 applications per page; Prev/Next when there are more

If you are filtered to one renewal year but pending applications exist in other years, a yellow banner appears with a link to Show all years so older pending items are not overlooked.

Payment display

The Payment (from website) section summarizes what WPForms collected:

  • Membership dues — from the membership type label (e.g. Adult - $80.00)
  • Initiation and Processing fee
  • Subtotal — when dues, initiation, and processing are all present
  • Special code — coupon or promo code; shows “(coupon applied)” when total paid is below subtotal
  • Total paid — from Total (Membership + Fees) on the form

Gateway and transaction fields appear when your automation sends them; many clubs only send fee line items and total.

Approve & reject

Approve

  • Matched member — updates only non-empty fields from the form (does not wipe existing data).
  • No match — creates a new member record.
  • Ambiguous match — pick the correct member from the dropdown before approving.

Reject

Rejecting moves the application off the Pending tab and keeps it under Rejected with an optional reason. Rejected rows remain in the database for reference.

Badge photo

When an applicant uploads a badge photo on the website (JPEG or PNG recommended), approving the application downloads that image from your WordPress site and saves it as the member's photo — the same file used for badge printing. You do not need to upload it again on the member record unless import failed.

  • Supported formats: JPEG, PNG, and GIF (same as manual member photo upload).
  • AMA / FAA files — review links only; they are not copied into RC Flight Operations.
  • Failed import — approval still succeeds; a yellow warning asks you to upload the photo on the member's Contact tab before printing.

Ask applicants to use Take photo on the form (especially on phones) and limit the WPForms field to .jpg, .jpeg, and .png for the smoothest experience.

Test submissions There is no delete button in the app yet. To fully remove a test application (for example so the same WPForms entry can be sent again), an admin must delete the row from the member_applications table in the database. Rejecting alone does not remove it.

Website setup

Connecting WPForms to RC Flight Operations requires a webhook secret in Administration → Installation and an Uncanny Automator recipe on your WordPress site. Full field mapping and curl test examples are in the repo guide WPFORMS_INTEGRATION.md (for club webmasters).
