Import & Export

Bring your existing member data in from a spreadsheet, and export your roster in multiple formats for newsletters, AMA filings, and backups.

CSV Import

If you're migrating from another system (a spreadsheet, an old database, another membership app), you can import members in bulk using a CSV file. Go to Members → Import (Editor or Admin role required).

Download the sample CSV on the import page to see the expected column format before preparing your own file.

Import Steps

  1. Prepare your CSV. Your file needs at minimum a first name and last name column. All other columns are optional. Comma or semicolon delimiters are both accepted. UTF-8 encoding (with or without BOM) is recommended. Max file size: 2 MB.
  2. Upload. On the Import page, choose your file and click Upload and map columns.
  3. Map columns. RC Flight Operations shows your CSV column headers and asks you to match each one to a member field. First name and last name are required. Leave "Don't map" for columns you don't want to import.
  4. Preview. You'll see a preview of the first several rows showing how the data will be mapped. Check for obvious errors before proceeding.
  5. Import. Click Import members. The system processes each row: new members are added; if "Update existing" is checked, the importer looks up a unique existing member using tiered matching (strictest first: first + last + email + birthday when those columns are mapped and filled; else first + last + email; then first + last only when exactly one member shares that name).

Import Tips

  • Existing member matching (when "Update existing" is checked) tries the strictest keys first: all of first, last, email, and birthday when those CSV columns are mapped and the row has email + a parsed birthday; otherwise first+last+email when email is present; otherwise first+last only if exactly one member in your club already has that name. Multiple matches are skipped (counted as ambiguous) and a new row may still be inserted — review the import summary.
  • Rows with no name (both first and last name empty) are silently skipped.
  • Membership type values are normalized — "Adult Member", "ADULT", and "adult" all map to "Adult".
  • Date formats: The importer accepts MM/DD/YYYY, M/D/YYYY, YYYY-MM-DD, and most common date formats. When in doubt, use YYYY-MM-DD.
  • Large imports: Files over 500 rows may time out on some shared hosts. Split into smaller batches if needed.
  • Photos cannot be imported via CSV. Upload photos individually on each member's Contact tab.

CSV Export

Go to Members → Export CSV (dropdown button) or Members → More export options. Also accessible from any report's export toolbar.

Export Formats

FormatColumns includedBest for
Full All member fields — same structure as the import file Backups, round-trip import to another RC Flight Operations instance, data analysis
Short First/Last name, Email, AMA number & expiry, FAA number & expiry, Gate key AMA charter renewal filings, field safety records
Email only Last name, First name, Email Newsletter or campaign tools (e.g. Sender.net). Includes members with a non-empty email — manage unsubscribes in your mailing platform, not in RC Flight Operations. Scheduled AMA/FAA reminders (send_reminders.php) honor transactional opt-out in Sender.net when configured under Administration → Installation (newsletter unsubscribes are separate).

Each format can be filtered by:

  • All members — every member in the database
  • Members for year — only members who renewed in a specific year
  • Not renewed for year — members who had last year's renewal but not the selected year

PDF Export

PDF export is available from the Reports page export toolbar. It uses Dompdf (free, open-source, MIT licensed) — no external service required. PDFs include the report title, stat cards summary, and the full data table.

PDF generation for very large reports (1,000+ rows) can be slow on shared hosting. For large datasets, CSV export is faster and more practical.
