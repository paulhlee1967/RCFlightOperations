Administration

Manage who can log in and what they can do, customize your club's branding, and configure dues rates.

Most of this section requires the Admin role. The Badge Designer is also accessible to Editors. Other roles won't see the Administration menu.

Users & Roles

Go to Administration → Users. This lists every account that can log into RC Flight Operations for your club. Each account has an email (used to log in), a display name, a role, and an active/inactive flag.

Users are separate from members — a board member who logs in to run reports has a user account, but may or may not also be in the member database.

Adding a User

  1. Go to Administration → Users → Add user.
  2. Enter their email address — this is what they use to log in.
  3. Enter their display name (shown in the top-right nav badge).
  4. Choose a role (see below).
  5. Set a temporary password — they should change it after first login.
  6. Make sure Active is checked.
  7. Save.

To deactivate a user (e.g., board member who has left), edit their account and uncheck Active. Their login is disabled but their account is kept for audit history.

Club Configuration

Go to Administration → Configuration. Settings are organized into three tabs: General, Design, and Dues Rules.

General Settings

SettingDescription
Club nameAppears in the navbar and on the login page
Club slugShort URL-friendly identifier — used internally; don't change after initial setup

Design Settings

SettingDescription
LogoUpload a PNG/JPEG — shown in the navbar next to the club name. Recommended: horizontal logo, max 300px wide, transparent background.
FaviconUpload a small square icon (ideally 32×32 or 64×64 PNG) — appears in browser tabs.
Primary colorMain accent color: nav bar background, buttons, links. Use a hex code (e.g., #6f7c3d).
Primary dark colorHover/active state of the primary color — usually 15–20% darker.
Background colorPage background — light gray (#f5f5f5) works well.
Muted colorUsed for borders, dividers, and subtle separators.
Text colorMain body text — should have high contrast with the background.
Changes to design settings take effect immediately on save. Do a hard refresh (Ctrl+Shift+R / ⌘+Shift+R) if you don't see the new colors right away.

Dues Rules

The Dues Rules tab lets you set rates per membership type. Each type (Adult, Youth, Senior, Spouse) has independent settings:

SettingDescription
Annual duesFull-year renewal amount in dollars
Initiation feeOne-time fee added to a new member's first payment (set to 0 if not applicable)
Late feeAdded when a renewal is recorded as "Late"
Prorate monthsMembers joining during the last N months of the year pay a prorated amount. Set to 0 to disable proration.

See Renewals — Configuring Dues for more detail on how proration is calculated.

Email Configuration

Outgoing mail is configured in config.php (SMTP or PHP mail()). The app uses it for Email report (one formatted message to an address you choose), Email members (individual messages per member from selected reports), and other features that send mail.

Member preferences: Each member has Allow email from the club on the Contact tab. When unchecked, they are excluded from report-based email sends, the “Email only” CSV export, and scheduled reminders run by your host (AMA/FAA expiry notices). Officers should still follow your club’s policies for any communication outside the app.

By default the app uses the server's built-in mail, which works on many hosts but sometimes lands in spam.

For reliable delivery, your club's webmaster needs to configure SMTP (outgoing mail) in the server configuration. Good free options for club volumes:

  • Brevo (formerly Sendinblue) — free up to 300 emails/day, SMTP included
  • Gmail SMTP — use a club Gmail account with an App Password
  • Your host's SMTP — most cPanel hosts provide an outgoing mail server

If you don't have access to the server config, ask whoever installed RC Flight Operations for your club to set up SMTP. Technical documentation in the project repository includes an example configuration for developers.
